# 課程藍圖（Course Blueprint）

## 1. 課程名稱與副標

**主標**：用 Python 打造你的股市新聞情報系統
**副標**：串接台股新聞 API × 官方重大訊息 × 熱度量化，每天自動產出盤前情報

備選名稱（A/B 測試用）：
- 《股市新聞自動化實戰：從 API 抓取到盤前報告》
- 《打造個人版財經資訊終端機：Python 股市新聞情報課》

## 2. 一句話賣點（Elevator Pitch）

> 法人有 Bloomberg、投顧有研究團隊，散戶只有滑不完的新聞 App。
> 這門課教你用 Python 把 10+ 家財經媒體與證交所官方公告接成一條自動化管線，
> 每天早上 7:30，你的電腦自動告訴你「今天哪些股票最熱、為什麼」。

## 3. 目標學員（Persona）

### 主要：「想升級的工程師散戶」（佔預估銷量 50%）
- 25–45 歲，軟體工程師或理工背景上班族，有台股帳戶
- 會 Python 基礎（會跑 script、裝套件），但沒做過爬蟲/資料管線
- 痛點：每天看新聞很花時間、資訊落後、不知道哪些新聞真的重要
- 買課動機：想要「自己做得出來」的成就感 + 實際省時間

### 次要：「投顧會員／訂閱戶」（佔預估銷量 30%，透過合作投顧導流）
- 已付費訂閱投顧服務，對「量化」「訊號」有基本認知
- Python 可能是零基礎 → 課程模組 1 要把環境建置講到保姆級
- 買課動機：投顧老師背書 + 想看懂訊號背後的邏輯

### 第三：「金融從業者／研究助理」（佔預估銷量 20%）
- 券商、投信投顧、財經媒體的年輕從業者
- 買課動機：工作上要蒐集整理新聞，想自動化

### 明確排除（銷售頁要講清楚，降低退費）
- 完全沒碰過程式、也不打算打開終端機的人（可引導先上免費的 Python 入門）
- 想要「明牌」或「保證獲利策略」的人——本課教的是資訊系統，不是投資建議

## 4. 學習成果（Learning Outcomes）

上完課，學員能夠：

1. 用 `requests` 串接鉅亨網公開 JSON API，抓取即時與歷史新聞
2. 用一個「註冊表模式」的通用 RSS 爬蟲，一次接入中央社、自由財經、MoneyDJ 等 7+ 家媒體
3. 串接 TWSE/TPEx OpenAPI，取得上市櫃公司官方重大訊息（第一手，不經媒體）
4. 處理真實世界的資料髒問題：時區正規化、兩層去重（同標題／改寫轉載）、HTTP 重試
5. 理解並實作 Point-in-Time（PIT）正確的儲存：`actionable_ts` vs `publish_ts`、假日感知
6. 建立全市場 ~1,800 檔的個股字典，把每則新聞自動標上相關股票代號
7. 計算個股新聞熱度分數、抽取券商目標價訊號、理解 Tier 分級與 veto 的設計思想
8. 把整條管線自動化：盤中監看事件串流 + 日終 ETL + 排程，產出每日盤前報告

**結業專題**：學員繳交一份「自己跑出來的盤前熱度報告」截圖 + 一段自訂功能（如新增一個新聞來源），完成者發結業證書。

## 7. 差異化（為什麼買這門而不是別門）

1. **不是玩具範例**：課程程式碼是一套真的在跑的系統，含 CI、測試、去重、PIT 儲存——別的爬蟲課教到「抓下來存 CSV」就結束了。
2. **官方資料源**：教 TWSE/TPEx OpenAPI 重大訊息，這是多數爬蟲課不會碰的第一手資料。
3. **量化思維**：不只抓資料，還教「怎麼判斷哪則新聞重要」（熱度分數、Tier、反向因子），這是投顧合作方能背書的專業層。
4. **抗改版承諾**：新聞網站會改版，課程承諾一年內維護範例程式碼。
