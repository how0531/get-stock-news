# 課程營運手冊：平台、金流、退費、法遵

> 開賣前必讀。法律段落為一般性整理，正式上線前建議請律師／合作投顧法遵部門過目一次。

---

## 1. 開課平台選擇

| 方案 | 抽成/費用 | 流量 | 適合情境 | 建議 |
|------|-----------|------|----------|------|
| **Teachify（開課快手）等 SaaS 自售** | 月費制＋金流手續費（~3%），營收幾乎全留 | 自帶流量＝0，全靠自己與投顧導流 | 已有導流來源 | ✅ **首選**：本案有投顧導流，不需平台流量；毛利最高、學員名單自有 |
| Hahow / PressPlay Academy | 抽成約 30–50%（依合約與導流歸屬） | 平台有站內流量與募資機制 | 沒有自有流量時 | 備選：若投顧導流不如預期，可第二波上架（注意平台可能要求獨家或最低價） |
| 自架（WordPress + 綠界/藍新） | 金流手續費 ~2–3% | 0 | 工程資源充足 | 不建議：維運成本吃掉省下的月費 |

**建議組合**：Teachify 類 SaaS 開站 + 綠界/藍新金流 + 自有 Email/LINE 名單。學員影音託管與防盜鏈由平台處理。

## 2. 金流與發票

- **收款方式**：信用卡（含分期，客單 2–3K 可開 3 期）、ATM 虛擬帳號、超商代碼——三種都開，超商對非工程師受眾（投顧會員）轉換影響大
- **發票**：個人賣課達營業規模應辦稅籍登記；建議一開始就以**行號/公司**名義開課，開立雲端發票（B2C 存載具、B2B 打統編）。與投顧的分潤需請款發票往來
- **對帳**：投顧折扣碼成交明細每月匯出 → 依分潤比例月結，保留報表存檔
- **定價顯示**：售價含稅；早鳥/專案價的原價需真實存在（定價 NT$2,980 需真的以此價販售過或同步掛牌），避免違反公平交易法的不實價格比較

## 3. 退費政策（合規重點）

**法規背景**：線上課程屬「通訊交易」，原則上適用消保法 7 天猶豫期（無條件解約）。但依《通訊交易解除權合理例外情事適用準則》，**「非以有形媒介提供之數位內容，一經提供即為完成之線上服務」**，經消費者事先同意者，可排除 7 天解約權。

**建議政策（銷售頁與結帳頁都要寫，結帳需勾選同意）**：

> 1. 購買後 7 天內且**未觀看任何正式課程內容**者，可申請全額退費。
> 2. 本課程為非以有形媒介提供之數位內容，一經開始觀看正式課程內容，即依《通訊交易解除權合理例外情事適用準則》不適用 7 天猶豫期，恕無法退費。您於結帳時勾選同意本條款。
> 3. 免費試看內容之觀看不影響退費資格。

**執行注意**：
- 「事先告知＋取得同意」是排除猶豫期的**生效要件**——結帳流程必須有明確勾選框，勾選紀錄要留存
- 平台後台需能查觀看紀錄作為依據
- 個案爭議以和為貴：退費糾紛上到消保官的成本遠高於一單課程費

## 4. 法遵紅線（與投顧合作最重要的一節）

### 4.1 證券投資顧問相關（證券投資信託及顧問法）

**核心原則**：對不特定大眾提供「個別有價證券之價值分析或買賣建議」屬證券投資顧問業務，須由持照投顧事業與合格人員為之。**本課程必須定位為「程式技術教學」**，具體守則：

1. 課程內容、行銷素材、直播、Discord 官方發言，**一律不得**出現：個股買賣建議、目標價評價、「這支可以關注/進場」等語句
2. 課程中的個股僅為「程式輸出範例」，錄影時遇到即時輸出要口頭聲明（模組 7 講稿已內建）
3. 每支影片片頭聲明卡：「本課程為程式與資料技術教學，非投資分析或投資建議」
4. Discord 版規明訂：禁止討論買賣建議；學員違規由管理員刪文提醒
5. **與投顧的分工**：投顧掛名「監製/方法論顧問」限於研究方法層面；若投顧要對其會員講「怎麼用這套系統的輸出做投資判斷」，那是**投顧以其執照身分、在其自身服務內**進行，與課程本體切開，文案不得混同
6. 所有含投顧名義的行銷素材，發出前送投顧法遵審閱並留存紀錄

### 4.2 著作權（新聞內容）

1. 課程教學示範以「標題＋連結＋發布時間＋量化統計」為主——事實與資料本身不受著作權保護，風險低
2. 系統雖可抓全文供**個人研究**內部處理，課程須明確教育學員：**不得重製全文對外散布或作為服務內容輸出**（repo README 已聲明，課程模組 1-3 專堂講授）
3. 錄影畫面出現新聞全文時，僅短暫示意、打碼或滾動帶過，不停留展示完整文章
4. 課程素材（講義、程式碼）為自有著作；購課條款註明僅供個人使用、禁止轉售散布

### 4.3 爬蟲與資料取得

1. 課程使用的取得方式：公開 JSON API、RSS、sitemap、官方 OpenAPI——皆為網站主動提供之機器可讀介面
2. 內建禮貌機制並在課程中強調：sleep 節流、盤中輪詢 ≥60 秒、指數退避、不繞過付費牆、不破解驗證
3. 若某來源明確以 robots.txt 或服務條款禁止，課程改用其他來源示範（註冊表模式讓替換成本極低）

### 4.4 個資（學員資料）

1. 收集 Email/LINE 名單需告知蒐集目的（課程通知與行銷），提供退訂機制
2. 與投顧交換的僅為「折扣碼成交統計」，不交換學員個資明細；若需交換需另取得學員同意
