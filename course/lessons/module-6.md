# 模組 6｜個股辨識：股票字典與新聞對應（講師教學筆記）

> 總長約 50 分鐘 / 4 堂。從「新聞管線」跨進「股市系統」的一步：每則新聞自動標上股票代號。

## 教學目標

- 建立與維護全市場股票字典（台股 ~1,800 檔 + 美日股）
- 理解中文個股辨識的陷阱（簡稱、撞名、產品名）
- 把 tickers 標註整合進管線

---

## 6-1 問題定義（10'）

1. 現況：儲存層裡有幾千則乾淨的新聞——但「哪則新聞講哪支股票」還不知道
2. 目標：每筆新聞加一個 `tickers` 欄位，如 `["2330", "2454"]`
3. 方法選型（誠實比較）：
   - 字典比對（本課）：可控、可解釋、快、離線 → 適合這個規模
   - NER/LLM：更聰明但成本高、不可控 → 列進階路線圖
4. 原則：**先用最笨但可靠的方法把系統跑起來，再局部升級**

## 6-2 建立全市場股票字典（14'）

**Demo**：`scripts/build_stock_dict.py` + `data/stocks/tw_stocks.json`
1. 資料來源：交易所公開的上市/上櫃公司清單
2. 跑 `--full-tw`：一次建好全市場 ~1,800 檔
3. 帶看字典結構：代號、正式名稱、常用簡稱、別名清單
4. 美日股字典（`us_stocks.json` / `jp_stocks.json`）：同樣結構，示範多市場擴充

## 6-3 比對的陷阱（14'）★本模組精華

用真實例子逐一講：
1. **簡稱歧義**：新聞不會寫「台灣積體電路製造」，只會寫「台積電」——字典必須收簡稱
2. **撞名災難**：
   - 「大同」：是公司名也是常用詞（大同小異）
   - 「長榮」：海運？航空？還是「長榮大學」？
   - 「精華」：光學股「精華」vs 一般用語「精華區」
3. 對策工具箱：
   - 黑名單/歧義名單：撞名嚴重的簡稱要求「代號同現」或上下文詞（如「股」「公司」）才算
   - 標題命中權重 > 內文命中（標題提到才是真的在講它）
   - 寧可漏掉、不要誤標：誤標會直接污染熱度分數
4. 觀念：字典是**活的**——新上市、改名、下市，每季要維護

## 6-4 管線整合（12'）

**Demo**：`process_day.py` 中 tickers 標註的位置
1. 流程定位：去重之後、寫入 processed 之前
2. 跑一次完整 `process_day.py`，抽幾筆看 `tickers` 結果：命中的、空的（總經新聞）、多檔的（產業稿）
3. 呼應模組 3 的原則：「`tickers` 為空的新聞我們不刪——過濾是下游的決策」
4. 收尾：資料面全部完工。下一模組開始「用這些資料算東西」
