# 模組 7｜熱度量化入門（講師教學筆記）

> 總長約 70 分鐘 / 6 堂。投顧合作方最能加值的模組；也是法遵最敏感的模組。
>
> ⚠️ **錄製紅線**：只講方法與程式，所有個股都以「範例輸出」呈現，不做任何買賣評價、
> 不說「可以買」「會漲」等語句。片頭放「本內容為程式教學，非投資建議」聲明卡。

## 教學目標

- 理解新聞熱度的正確定義：邊際變化 > 絕對熱度
- 會跑 v1 快速熱度，理解評分公式的設計取捨
- 會抽取券商目標價訊號
- 理解 Tier 分級、veto、反向因子與事後驗證的設計思想

---

## 7-1 熱度的定義（12'）★免費試看堂

**講稿大綱**
1. 反例開場：「台積電今天 30 則新聞，算熱嗎？」——不算，它天天 30 則
2. 正例：「某中小型股平常 0–1 則，今天 8 則」——這才是訊號。**邊際變化比絕對熱度重要**
3. 統計語言：z-score——今天的量離它「自己的常態」幾個標準差
4. 第二原則（README 三方專家共識）：**新聞是落後指標、籌碼是領先指標**——熱度告訴你「市場在看哪」，不告訴你「該不該買」
5. 誠實聲明（原話錄進去）：「這個模型量化的是關注度，不是漲跌預測。怎麼用它，是你自己的投資決策。」

## 7-2 v1 快速熱度（12'）

**Demo**：`skills/stock-heat-model/scripts/quick_heat.py`
1. 公式極簡版：標題命中 ×2 ＋ 摘要命中 ×1 ＋ 雜訊過濾
2. 逐項講設計理由：
   - 標題權重高：記者把重點放標題，標題提到＝真的在講它
   - 雜訊過濾：盤勢綜述類文章（「台股收漲百點」）會提到一堆股票，全都要降權
3. 現場跑一天資料，看 Top 10 輸出
4. 定位誠實講：v1 是 sanity check 用的簡化版；v2 完整公式（多因子）是進階課範圍——但 v1 已經能每天給你有用的排序

## 7-3 券商目標價訊號抽取（14'）

**Demo**：`skills/stock-heat-model/scripts/extract_target_price.py`
1. 素材：Factset 目標價速報 + 一般新聞裡的「目標價上看 X 元」句式
2. regex 實戰（本課的正則教學段）：抓「目標價」「上看」「調升至」等句式 + 金額
3. 輸出 `data/target_price/YYYY-MM-DD.json`：日期、代號、券商、目標價、方向
4. 陷阱提醒：同一篇稿可能提多檔目標價；「目標價 vs 現價」要配價格資料才有意義（進階路線圖）

## 7-4 Tier 分級與 veto（12'）

照 `skills/stock-heat-model/SKILL.md` 講設計思想（不 demo 完整實作，v2 屬進階課）：
1. Tier S/A/W/X：把連續分數變成可執行的離散等級——S（強訊號）/ A（關注）/ W（觀察）/ X（排除）
2. veto 規則：一票否決制——例如已被列注意/處置股票、或籌碼面背離時，熱度再高也降級
3. 設計哲學：「加分項讓你找到標的，**否決項讓你活下來**」
4. 法遵句（錄進去）：Tier 是資訊整理的分級，不是投資評等

## 7-5 反直覺因子：expert_repeat（10'）

1. 現象：投顧專欄「連推同一支 3 天」→ 統計上後續報酬反而差
2. 機制解釋：連推多天＝散戶關注已擁擠＝該進場的都進了——利多出盡的結構
3. 所以 `expert_repeat` 在模型裡是**反向因子**（README 原則：連推 3 天 → 訊號反轉）
4. 教學價值：因子方向不能靠直覺，要靠 7-6 的事後驗證來定

## 7-6 事後驗證：T+1/T+5/T+20（10'）

1. 流程：每天記錄 Tier 名單 → 對齊 T+1/T+5/T+20 報酬 → 定期檢視因子是否仍有效
2. 這裡呼應模組 5 的 PIT：**驗證用 `actionable_ts` 對齊**，否則整套驗證是自欺
3. 觀念：模型會退化（regime change）——驗證不是一次性作業，是例行維運
4. 收尾：資料 → 分數 → 分級 → 驗證，量化閉環完成。最後一個模組：讓它全自動跑
