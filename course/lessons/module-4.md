# 模組 4｜官方第一手：TWSE/TPEx 重大訊息 OpenAPI（講師教學筆記）

> 總長約 45 分鐘 / 4 堂。差異化模組——多數爬蟲課不教官方資料源，這是投顧合作方最能背書的一段。

## 教學目標

- 理解「資訊鏈源頭」的價值：公司公告先於所有媒體報導
- 會用 TWSE / TPEx OpenAPI 抓上市櫃重大訊息
- 理解官方資料與媒體新聞如何互補合流

---

## 4-1 為什麼官方公告比媒體快又準（10'）

**講稿大綱**
1. 資訊鏈複習（模組 1-2 的圖）：重大訊息是**源頭**——公司依法必須先在公開資訊觀測站發布，媒體才據此寫稿
2. 三個優勢：
   - **快**：領先媒體數分鐘到數小時
   - **準**：法定格式、無記者轉譯誤差
   - **全**：小型股的公告媒體根本不會寫，但公告一定有
3. 實例（用歷史案例，不做任何買賣評價）：某公司「處分資產」公告 → 隔天媒體才出報導
4. 限制誠實講：OpenAPI 是「當日快照」非逐秒推播；秒級要靠 MOPS 即時端點（反爬較強，列進階路線圖）

## 4-2 TWSE OpenAPI 實戰（12'）

**Demo**：打開 `scripts/twse_announce.py`
1. 端點：TWSE OpenAPI `t187ap04_L`（上市公司當日重大訊息）——**這是官方文件化的 open data，連 headers 都不用偽裝**
2. 現場跑：
   ```bash
   PYTHONUTF8=1 python scripts/twse_announce.py
   ```
3. 帶看欄位：公司代號、名稱、發言日期/時間、主旨
4. 對照課程標準化格式：官方公告也轉成與新聞相同的欄位結構（source=`announce`），下游一視同仁
5. 順帶介紹 TWSE OpenAPI 目錄（openapi.twse.com.tw）：「這裡還有幾百個端點——注意股票、融資融券、三大法人……都是免費的」

## 4-3 TPEx OpenAPI：上櫃重大訊息（10'）

1. 上市（TWSE）與上櫃（TPEx）是兩個機構、兩套 API：`mopsfin_t187ap04_O`
2. Demo：同一支 script 同時抓兩邊，合併輸出
3. 提醒：中小型上櫃股媒體覆蓋率低 → **官方公告是上櫃股幾乎唯一的即時資訊源**，價值反而更高

## 4-4 官方資料與媒體新聞的互補（12'）

**講稿大綱**
1. 時間軸圖：公告（T+0 分）→ 通訊社快訊（T+10 分）→ 各媒體改寫（T+30 分～數小時）→ 產業深度稿（T+1 天）
2. 合流設計：公告與新聞進同一條事件串流，靠 `source` 欄位區分權重（公告可信度最高）
3. 「媒體報導量」本身是訊號：同一件公告，有 5 家媒體跟進 vs 0 家跟進，市場關注度完全不同 → 這正是模組 7 熱度模型的 `cross_media` 因子
4. 收尾：至此抓取層完工（API + RSS + sitemap + HTML + 官方 OpenAPI 五種武器都有了），下一模組進儲存層

## 作業 4

抓出今天所有上市＋上櫃重大訊息，篩出主旨含「處分」或「取得」資產相關的公告，輸出公司名稱與主旨。
**評分標準**：同時包含 TWSE 與 TPEx 兩邊資料。
**常見卡關**：假日跑作業（當日無公告）→ 提醒平日跑，或接受空結果 + 說明原因。
