# 售前 FAQ 與客服 SOP

> 前半是放在銷售頁/購買頁的公開 FAQ；後半是客服（或你自己）回覆用的內部 SOP。

## 公開 FAQ

### 課程內容

**Q1：我需要什麼程度的 Python？**
只要你曾經成功執行過 Python 程式、用 pip 裝過套件即可。課程用到的語法不超過：函式、字典/串列、迴圈、try/except。模組 1 有 Windows 與 Mac 各一支保姆級環境建置影片。

**Q2：完全零基礎可以買嗎？**
不建議直接買。請先完成任何一個免費 Python 入門（約 6–10 小時），能自己跑出 `print("hello")` 和 `pip install requests` 再來，學習體驗會好非常多。

**Q3：課程會教怎麼選股／預測漲跌嗎？**
不會。本課程是**程式與資料工程教學**，教你建立新聞資訊系統與量化「市場關注度」；不提供任何證券投資分析或買賣建議，課程中出現的個股皆為程式輸出範例。投資有風險，決策請自行判斷。

**Q4：跟一般爬蟲課差在哪？**
三點：(1) 教官方資料源（TWSE/TPEx OpenAPI），不只媒體爬蟲；(2) 教資料工程硬功夫——兩層去重、時區正規化、Point-in-Time 儲存；(3) 附一整套含測試與 CI 的生產級 GitHub 專案，不是課堂片段程式碼。

**Q5：Mac / Windows 都可以上嗎？**
可以，兩個平台的環境建置與排程設定都有各自的完整示範。需要 Python 3.11 以上。

**Q6：課程觀看期限？**
購買後永久觀看，含未來的課程內容更新。

### 程式碼與維護

**Q7：新聞網站改版、程式跑不動怎麼辦？**
兩層保障：(1) 課程本身就教你「來源健檢與維修」（healthcheck、來源結構地圖、註冊表模式）；(2) 購課後一年內，因來源改版導致課程範例失效，我們免費更新程式碼並發布公告。

**Q8：程式碼可以商用嗎？**
程式碼供學員個人學習與個人投資研究使用。請勿轉售課程內容或將抓取之新聞全文重製散布（著作權課堂會詳細說明界線）。

**Q9：需要付費的 API key 或訂閱嗎？**
不用。課程使用的資料源全部免費：公開 JSON API、RSS、sitemap、證交所 OpenAPI。

### 購買與退費

**Q10：付款方式？**
信用卡／ATM 轉帳／超商代碼（依開課平台支援為準）。

**Q11：可以退費嗎？**
可以。購買後 7 天內且未觀看正式課程內容（免費試看不計），可申請全額退費。一旦開始觀看正式內容，因數位內容特性恕不退費（購買前會請你勾選同意）。詳見購買頁退費政策。

**Q12：可以開發票／報帳嗎？**
可以，結帳時可填統編開立雲端發票。

## 內部客服 SOP

### 回覆時效承諾
- 售前問題：24 小時內
- 學員卡關（Discord `#卡關求救`）：工作日 24 小時內；環境建置類優先處理
- 退費申請：3 個工作日內完成

### 高頻情境回覆模板

**情境 A：問「上完課能不能賺錢／有沒有推薦標的」**
> 謝謝你的興趣！先說清楚：這門課教的是「用程式自動蒐集與量化新聞資訊」的技術，不提供投資建議，也不會推薦任何標的。如果你期待的是選股訊號服務，這門課可能不適合你，建議先看免費試看三堂再決定。
（原則：主動勸退，寧可少賣一單，不收錯期待的學員。）

**情境 B：零基礎詢問**
> 課程需要一點點 Python 基礎（能執行程式、會 pip 安裝）。如果還沒碰過，推薦先完成【免費入門資源連結】，大約一個週末的時間，之後再來上這門課會順很多。可以先收藏早鳥資訊，我們開賣期【日期】前都來得及。

**情境 C：買錯／重複購買**
> 直接協助辦理退費或轉單，3 個工作日內處理，不需理由。

**情境 D：退費但已觀看**
> 依購買時同意之政策，已觀看正式內容恕無法退費。但請告訴我們卡在哪裡——如果是內容問題我們想修正；若是環境問題，Discord 有專人協助你跑起來。
（升級條款：若學員因課程內容錯誤導致無法完成，個案專人處理，必要時破例退費——口碑優先。）

**情境 E：程式跑不動**
> 請提供：(1) 作業系統、(2) `python --version` 輸出、(3) 完整錯誤訊息截圖、(4) 執行的指令。貼到 Discord `#卡關求救`，教學助理會接手。
（內部：常見錯誤對照表見模組 1 講義附錄；新增的常見問題每月回填到講義。）

### 退費處理流程
1. 收到申請 → 查後台觀看紀錄
2. 未觀看正式內容 → 平台後台執行退費 → 回覆確認信
3. 已觀看 → 套用情境 D 模板 → 視個案升級
4. 每月統計退費原因，回饋到課程內容與銷售頁「不適合誰」段落
